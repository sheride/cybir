from dataclasses import dataclass, field
import numpy as np

@dataclass
class ToricCurveData:
    """Container for toric curve computation results.

    Attributes
    ----------
    flop_curves : list of numpy.ndarray
        Curves classified as flop-type from toric data.
    weyl_curves_g0 : list of numpy.ndarray
        Weyl-type curves at genus 0.
    weyl_curves_higher_genus : list of numpy.ndarray
        Weyl-type curves at higher genus.
    other_curves : list of numpy.ndarray
        Curves not fitting flop or Weyl categories.
    minface1_curves : list of numpy.ndarray
        Curves from 1-face minimal edges.
    gv_dict : dict
        Map from curve tuple -> toric GV invariant.
    """
    flop_curves: list = field(default_factory=list)
    weyl_curves_g0: list = field(default_factory=list)
    weyl_curves_higher_genus: list = field(default_factory=list)
    other_curves: list = field(default_factory=list)
    minface1_curves: list = field(default_factory=list)
    gv_dict: dict = field(default_factory=dict)

    def merge(self, other):
        """Merge another ToricCurveData into this one (in place)."""
        self.flop_curves.extend(other.flop_curves)
        self.weyl_curves_g0.extend(other.weyl_curves_g0)
        self.weyl_curves_higher_genus.extend(other.weyl_curves_higher_genus)
        self.other_curves.extend(other.other_curves)
        self.minface1_curves.extend(other.minface1_curves)
        self.gv_dict.update(other.gv_dict)

    def all_curves(self):
        """Return all curves as a flat list."""
        return (self.flop_curves + self.weyl_curves_g0 +
                self.weyl_curves_higher_genus + self.other_curves +
                self.minface1_curves)

```python
def classify_phase_type(kahler_cone, charge_matrix, moving_cone_obj=None):
    """Classify a phase as FRST, vex, or non-inherited.

    Parameters
    ----------
    kahler_cone : cytools.Cone
        The phase's Kahler cone.
    charge_matrix : numpy.ndarray
        GLSM charge matrix Q, shape (h11, n_points).
    moving_cone_obj : cytools.Cone, optional
        Pre-computed moving cone. If None, computed from charge_matrix.

    Returns
    -------
    tuple of (str, object)
        (phase_type, fan_or_none) where phase_type is 'frst', 'vex',
        or 'non_inherited', and fan_or_none is the regfans Fan for
        frst/vex phases or None.

    Notes
    -----
    Per D-06 in CONTEXT.md. Uses regfans VectorConfiguration for FRST
    detection. Uses CYTools Cone.intersection() and Cone.is_solid()
    for checking solid intersection with the moving cone (both methods
    verified to exist on the CYTools Cone class).
    """
    from regfans import VectorConfiguration
    from .util import moving_cone

    if moving_cone_obj is None:
        moving_cone_obj = moving_cone(charge_matrix)

    # Check solid intersection with moving cone
    try:
        intersection = kahler_cone.intersection(moving_cone_obj)
        if not intersection.is_solid():
            return ("non_inherited", None)

        # Find interior point and lift to heights
        J = intersection.tip_of_stretched_cone(1)
    except Exception:
        return ("non_inherited", None)

    # Lift to height vector: solve Qx = J (least squares)
    Q = np.asarray(charge_matrix, dtype=float)
    h = np.linalg.lstsq(Q, J.astype(float), rcond=None)[0]

    # Subdivide vector configuration
    vc = VectorConfiguration(Q.T)
    try:
        fan = vc.triangulate(heights=h)
    except Exception:
        return ("non_inherited", None)

    if fan.respects_ptconfig():
        return ("frst", fan)
    else:
        return ("vex", fan)
```

```python
def orient_curves_for_phase(curves, tip):
    """Re-orient toric curves for a specific phase.

    Each curve is oriented so that tip @ curve > 0.

    Parameters
    ----------
    curves : list of numpy.ndarray
        Curves in canonical orientation.
    tip : numpy.ndarray
        Interior point of the phase's Kahler cone.

    Returns
    -------
    list of numpy.ndarray
        Re-oriented curves.
    """
    oriented = []
    for c in curves:
        sign = np.sign(tip @ c)
        if sign == 0:
            oriented.append(c)
        else:
            oriented.append(int(sign) * c)
    return oriented
```

1. **Port `compute_toric_curves` function** to `toric_curves.py`. This is the main toric curve computation, ported from `compute_toric_curves_old` in cornell-dev. Split into enumeration and diagnosis:

```python
def compute_toric_curves(cy, face_triangulations, tip=None):
    """Compute toric curves from induced 2-face triangulations.

    Enumerates edges in the 2-face triangulations, computes their
    charge vectors in the GLSM basis, classifies by normal bundle
    and enveloping divisor position, and diagnoses as flop/Weyl/other.

    Parameters
    ----------
    cy : cytools.CalabiYau
        The Calabi-Yau threefold.
    face_triangulations : list of list of numpy.ndarray
        Induced 2-face triangulations (from induced_2face_triangulations).
    tip : numpy.ndarray, optional
        Interior point of the Kahler cone for orientation.

    Returns
    -------
    ToricCurveData
        Classified toric curves with GV invariants.

    Notes
    -----
    Ported from cornell-dev/lib/geom/curves/toric.py function
    ``compute_toric_curves_old``. CRITICAL: uses
    ``intersection_numbers(in_basis=False)`` for raw point indices.
    """
```

The port should follow the exact algorithm from cornell-dev:

a. Get polytope and geometric data:
   - `polytope = cy.polytope()`
   - `int_nums_raw = cy.intersection_numbers(in_basis=False, format="dense")` -- CRITICAL: in_basis=False
   - Get 2-faces: `twofaces = [set(f.points(as_indices=True)) for f in polytope.faces(2)]`
   - Get 1-faces: `onefaces = [set(f.points(as_indices=True)) for f in polytope.faces(1)]`
   - Get vertices: `vertices = set(f.points(as_indices=True)[0] for f in polytope.faces(0))`

b. For each 2-face and each triangulation of that face:
   - Find all edges (pairs of points in simplices)
   - Classify edges as "twoface" (shared by 2 triangles) or "oneface" (in 1 triangle)
   - For twoface edges: compute double intersection number from the two triangles
   - For oneface edges: these sit on the boundary

c. For each edge, compute:
   - Normal bundle from double intersection numbers (reversed signs)
   - Enveloping divisors (which faces contain the edge endpoints)
   - Charge vector in GLSM basis using `cy.glsm_charge_matrix(include_origin=False)`
   - Toric GV invariant from normal bundle

d. Classify into categories:
   - Flop curves: normal bundle (-1, -1)
   - Weyl genus-0: specific normal bundle patterns
   - Weyl higher-genus: specific patterns
   - Other: everything else

e. Orient curves using tip if provided (per D-10).

**IMPORTANT:** Read the full `compute_toric_curves_old` source (all ~698 lines) before implementing. The function has many edge cases and the exact algorithm matters for correctness. Do NOT simplify or skip edge case handling.

f. If the cornell-dev source is not accessible, ask the user for the source file location. Do NOT attempt to reconstruct the algorithm from memory or description alone.
