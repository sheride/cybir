```python
"""Weyl orbit expansion for the hyperextended Kahler cone.

Applies symmetric-flop Coxeter reflections to fundamental-domain
phases to discover additional phases beyond the fundamental domain.

See arXiv:2212.10573 Section 4.3 for the Weyl orbit expansion
algorithm.
"""

import logging
import numpy as np
from .types import CalabiYauLite, ExtremalContraction, ContractionType
from .util import normalize_curve, tuplify

logger = logging.getLogger("cybir")
```

Main function `expand_weyl(ekc)`:
1. Collect fundamental-domain phases: `fund_phases = list(ekc._graph.phases)` (all phases before expansion)
2. Collect symmetric-flop reflections: `reflections = [np.array(r) for r in ekc._sym_flop_refs]`
3. If no reflections: `logger.info("No symmetric-flop reflections; skipping Weyl expansion"); return`
4. `phase_counter = ekc._graph.num_phases` (continue numbering from BFS)
5. For each fundamental phase, for each reflection matrix M:
   a. Call `_reflect_phase(phase, M)` to create reflected phase data
   b. Check if reflected phase is new (compare Mori cone generators against all existing phases)
   c. If new: create `CalabiYauLite` with label `f"CY_{phase_counter}"`, add to graph
   d. Create an `ExtremalContraction` of type `SYMMETRIC_FLOP` linking the original and reflected phases
   e. `ekc._graph.add_contraction(contraction, phase.label, new_label)`
   f. `ekc._weyl_phases.append(new_label)`
   g. Inherit wall classifications: for each contraction from the original phase, the reflected phase should have a corresponding contraction with the same type. The reflected wall curve is `M @ original_curve`.
   h. `phase_counter += 1`
6. Log summary

Helper `_reflect_phase(phase, reflection_matrix)`:
- Compute `M = np.array(reflection_matrix)`
- `new_int_nums = np.einsum('abc,xa,yb,zc', phase.int_nums, M, M, M)` -- transform intersection numbers
- `new_c2 = M @ phase.c2` -- transform second Chern class (using einsum: `np.einsum('a,xa', phase.c2, M)`)
- Transform Kahler cone: `import cytools; new_kc = cytools.Cone(rays=phase.kahler_cone.rays() @ np.round(M).astype(int))`
- `new_mori = new_kc.dual()`
- Return `(new_int_nums, new_c2, new_kc, new_mori)`

Helper `_is_new_phase(ekc, mori_cone)`:
- Compare Mori cone generators (as sorted tuples of tuples) against all existing phases
- Return True if no match found

**CRITICAL**: The einsum for int_nums transformation uses indices `'abc,xa,yb,zc'` (same as original `sym_flop_cy`). The c2 transformation is `M @ c2` (matrix-vector multiply). Read the original carefully -- the convention matters for correctness.
