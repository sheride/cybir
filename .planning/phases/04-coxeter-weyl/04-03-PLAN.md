**Add to coxeter.py (per D-15, D-16):**

1. `to_fundamental_domain(point, reflections, curves, max_iter=1000)` -- Chamber walk algorithm.
   - `point`: ndarray in Mori space
   - `reflections`: list of symmetric-flop reflection matrices
   - `curves`: list of contraction curves (same order as reflections), defining the walls
   - Algorithm: repeatedly scan curves; if `point @ curve < 0`, reflect through that wall (`point = M @ point`, `g = M @ g`), restart scan. If no negative pairings found, return `(point, g)`.
   - Returns `(mapped_point, group_element)` -- both the fundamental-domain point AND the group element that maps it back (Claude's discretion: return both).
   - Safety: raise RuntimeError after max_iter reflections.
   - Docstring cites arXiv:2212.10573 Section 4.3.

2. `_invariants_for_impl(ekc, phase_label)` -- Helper for on-demand GV reconstruction (per D-15).
   - Get the phase from graph
   - Compute tip point via `_compute_tip` (import from build_gv)
   - Get root invariants from `ekc._root_invariants`
   - Find all FLOP-type contraction curves across ALL fundamental-domain phases
   - For each flop curve, check if `tip @ curve < 0` (in the Mori pairing sense)
   - Collect the negatively-pairing curves
   - Call `root_invariants.flop_gvs(negative_curves)` to get reoriented GVs
   - Return the reoriented Invariants object
   - NOTE: This relies on CYTools Invariants having a `flop_gvs` method (patched in Phase 3). If the method signature differs, adapt accordingly.
