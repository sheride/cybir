Fix SC-4: Reflected phases from `reflect_phase_data` have `curve_signs=None`, so `invariants_for()` silently returns root invariants.

The fix goes in `apply_coxeter_orbit` (not `reflect_phase_data` itself, because the curve_signs computation needs the full set of known flop curves from the ekc, which reflect_phase_data does not have access to).

**In `cybir/core/coxeter.py`, inside `apply_coxeter_orbit`:**

1. After creating the reflected phase at line 813 (`new_phase = reflect_phase_data(fund_phase, g, label=new_label)`), compute and attach `curve_signs` and `tip`:

   ```python
   new_phase = reflect_phase_data(fund_phase, g, label=new_label)

   # Compute curve_signs for reflected phase (SC-4 gap fix)
   # The tip of the reflected phase is g^{-T} @ fund_tip (Kahler space, D-08)
   # But for sign computation we need tip @ curve in Mori space,
   # which is equivalent to fund_tip @ g^{-1} @ curve.
   # Since we already have g_inv_int, use it.
   if fund_phase.tip is not None:
       reflected_tip = (g_inv_int.T @ fund_phase.tip.astype(float))
       new_phase._tip = reflected_tip

       # Collect all known flop curves from graph edges
       # (both fundamental and already-reflected)
       if fund_phase.curve_signs is not None:
           new_signs = {}
           for curve_tuple in fund_phase.curve_signs:
               # Reflect the curve: g @ curve (Mori space, D-08)
               reflected_curve = tuple(int(x) for x in g @ np.array(curve_tuple))
               # Sign = sign(reflected_tip . reflected_curve_in_original_basis)
               # But simpler: sign is preserved/flipped based on geometry.
               # Actually, use the same pattern as BFS:
               # sign = sign(tip @ curve) for each curve in the known set
               new_signs[curve_tuple] = int(np.sign(reflected_tip @ np.array(curve_tuple)))
           new_phase._curve_signs = new_signs
   ```

   Wait -- this needs care. The curve_signs dict keys are the set of ALL known flop curves discovered during BFS (in original Mori coordinates). Each phase's curve_signs maps each such curve to +1 or -1 based on `sign(tip @ curve)`. For reflected phases, the tip is `g_inv.T @ fund_tip` (the tip in the reflected Kahler cone). The sign for curve `c` is `sign(reflected_tip @ c)`.

   The root phase has curve_signs with keys = all curves discovered during BFS. The fundamental phases created during BFS also have these same keys. For reflected phases, we should use the SAME key set (the root phase's curve_signs keys) and compute signs using the reflected tip.

   Corrected approach:
   ```python
   if fund_phase.tip is not None:
       reflected_tip = g_inv_int.T.astype(float) @ fund_phase.tip.astype(float)
       new_phase._tip = reflected_tip

       # Use root phase's curve_signs keys as the canonical curve set
       root_phase = ekc._graph.get_phase(ekc._root_label)
       if root_phase.curve_signs is not None:
           new_signs = {
               c: int(np.sign(reflected_tip @ np.array(c)))
               for c in root_phase.curve_signs
           }
           new_phase._curve_signs = new_signs
   ```

   This is O(n_curves) per reflected phase, which is fine.
