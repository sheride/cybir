Create `tests/generate_ground_truth.py` -- the comprehensive ground truth generation script per D-01, D-03, D-04.

**Architecture:**
- Single script, `--h11 2` or `--h11 3` or `--h11 all` argument
- `--start N` / `--limit N` for resumable runs
- `--cache-dir` for GV cache location (default: `tests/ground_truth/gv_cache`)
- `--skip-full-bfs` to skip pass 2 (faster debugging)
- `--timeout N` per-polytope timeout (seconds, default 180)

**Core functions:**

1. `generate_one(poly_idx, polytope, h11, cache_dir, skip_full_bfs, timeout)`:
   - Run as subprocess (per Pitfall 3 -- CYTools multiprocessing deadlock in shells)
   - Actually, the subprocess pattern is for the outer loop. The inner function is:

2. `_generate_one_inner(poly_idx, polytope, h11, cache_dir, skip_full_bfs)`:
   - **Step A: Load/compute GVs**
     - Check `{cache_dir}/h11_{h11}/poly_{poly_idx}.pkl` for cached Invariants
     - If not cached: `cy = polytope.triangulate().get_cy()`, compute GVs via cybir `from_gv`, cache the `_root_invariants` as pickle
     - If cached: load pickle

   - **Step B: Run cybir fundamental domain** (for finite-type detection per D-04)
     - `ekc_cybir = CYBirationalClass.from_gv(cy, max_deg=10, verbose=False, gvs=cached_gvs)`
     - Extract `coxeter_type`, `coxeter_order`, whether sym_flop_refs exist, whether finite

   - **Step C: Run original code fundamental domain (pass 1)**
     - Import cornell-dev: `sys.path.insert(0, "/Users/elijahsheridan/Research/string/cytools_code/cornell-dev/projects/vex/elijah")`
     - `from extended_kahler_cone import ExtendedKahlerCone, CY_GV`
     - Create `ekc_orig = ExtendedKahlerCone(polytope)`
     - Inject shared GVs (same pattern as `compare_orbit.py:run_original_full`)
     - `ekc_orig.construct_phases(weyl=False, ignore_sym=True, verbose=False, limit=100)`
     - Extract fundamental domain data via `_extract_json()` and `_extract_npz()`

   - **Step D: Run original code full BFS (pass 2, only if finite Coxeter)**
     - Only if cybir detected finite Coxeter group (positive-definite bilinear form)
     - Fresh `ekc_orig2 = ExtendedKahlerCone(polytope)`, inject same GVs
     - `ekc_orig2.construct_phases(weyl=False, ignore_sym=False, verbose=False, limit=500)`
     - Extract full BFS data, add to JSON under `"full_bfs"` key

   - **Step E: Save JSON + npz**
     - `tests/ground_truth/h11_{h11}/poly_{poly_idx}.json`
     - `tests/ground_truth/h11_{h11}/poly_{poly_idx}.npz`

3. `_extract_json(poly_idx, h11, ekc_obj, pass_name, cybir_info=None)`:
   Return dict per the schema from RESEARCH.md Pattern 1:
   ```python
   {
       "polytope_idx": poly_idx,
       "h11": h11,
       "pass": pass_name,  # "fundamental" or "full_bfs"
       "n_phases": len(ekc_obj.cys),
       "has_finite_coxeter": cybir_info["has_finite_coxeter"],
       "coxeter_type": cybir_info["coxeter_type"],  # from cybir
       "coxeter_order": cybir_info["coxeter_order"],  # from cybir
       "n_sym_flop_refs": len([w for w in ekc_obj.walls if w.category == "symmetric flop"]),
       "n_infinity_gens": len(ekc_obj.infinity_cone_gens),
       "n_eff_gens": len(ekc_obj.eff_cone_gens),
       "n_unresolved": len([w for w in ekc_obj.walls if w.category is None]),
       "phase_labels": [f"phase_{i}" for i in range(len(ekc_obj.cys))],
       "per_phase_contraction_types": {
           f"phase_{i}": [_map_category(w.category) for w in cy_gv.walls if w.category is not None]
           for i, cy_gv in enumerate(ekc_obj.cys)
       },
       "contraction_type_counts": dict(Counter(
           _map_category(w.category) for w in ekc_obj.walls if w.category is not None
       )),
       "infinity_cone_gens": sorted([list(int(x) for x in g) for g in ekc_obj.infinity_cone_gens]),
       "eff_cone_gens": sorted([list(int(x) for x in g) for g in ekc_obj.eff_cone_gens]),
   }
   ```
   When pass 2 data exists, nest it under `"full_bfs"` key in the same JSON.

4. `_extract_npz(ekc_obj, pass_name_prefix="")`:
   Return dict of numpy arrays:
   - Per phase `i`: `{prefix}phase_{i}_int_nums`, `{prefix}phase_{i}_c2`, `{prefix}phase_{i}_kahler_rays` (sorted lexicographically), `{prefix}phase_{i}_tip`
   - Per wall `j` of phase `i`: `{prefix}phase_{i}_contr_{j}_curve`, `{prefix}phase_{i}_contr_{j}_zvd` (if exists)
   - Global: `{prefix}coxeter_refs` (stacked as 3D array), `{prefix}sym_flop_curves` (stacked)
   - For pass 2 prefix is `"full_"`. Both passes go into the same npz.

   **Critical: cast all data through int64** to avoid float contamination (Pitfall 1).
   **Critical: sort Kahler rays lexicographically** before storing (Pitfall 2).

5. `_map_category(category_str)`:
   Map original string categories to standard names:
   ```python
   {"asymptotic": "ASYMPTOTIC", "CFT": "CFT", "su(2) enhancement": "SU2",
    "symmetric flop": "SYMMETRIC_FLOP", "flop": "FLOP"}
   ```

6. `_inject_gvs(ekc_obj, gvs)`:
   Same pattern as `compare_orbit.py:run_original_full` lines 53-64:
   ```python
   cy = ekc_obj.polytope.triangulate().cy()
   gvs_copy = gvs.copy()
   gvs_copy.flop_curves = []
   gvs_copy.precompose = np.eye(cy.h11())
   from extended_kahler_cone import CY_GV
   ekc_obj.root = CY_GV(cy=cy, gvs=gvs_copy, cob=ekc_obj.cob)
   ekc_obj.root.toric = True
   ekc_obj.gvs = gvs_copy
   ekc_obj.walls.extend(ekc_obj.root.walls)
   ekc_obj.root.curve_signs = dict()
   ekc_obj.cys.append(ekc_obj.root)
   ```

7. `main()`:
   - Parse args, enumerate favorable polytopes via `fetch_polytopes(h11=N, lattice="N")`
   - Filter to favorable via `p.is_favorable("N")`
   - Loop with incremental progress printing
   - Each polytope: call `_generate_one_inner` (or subprocess wrapper if needed)
   - Print summary at end

**Key points:**
- Use `int()` and `np.asarray(..., dtype=np.int64)` everywhere to avoid float contamination
- Sort set-based data (infinity_cone_gens, eff_cone_gens) for deterministic JSON
- Kahler rays from `cy.mori.dual().rays()` must be sorted lexicographically
- Phase matching is by index (0, 1, 2...) since the original code has no labels
- The Coxeter type/order come from cybir (since the original code does not compute them)
- GV cache uses pickle (pragmatic exception per RESEARCH.md A1) -- these are transient test infrastructure

**Subprocess pattern for the outer loop (following run_compare_orbit.py):**
The main function should run each polytope as a subprocess call to itself with `--single POLY_IDX` mode, using `subprocess.run` with a timeout. This prevents CYTools multiprocessing deadlocks and allows per-polytope timeouts. The `--single` mode runs `_generate_one_inner` directly and writes to the output files.
