  <action>
Add 5 functions to the END of `cybir/core/util.py` (after `projection_matrix`). All per D-02, D-04, D-06, D-12.

1. `projected_int_nums(int_nums, curve, n_projected=3)`:
   - Uses `projection_matrix(curve)` to get projector P (shape (h11-1, h11))
   - Contracts `int_nums` with P along `n_projected` indices via sequential `np.einsum`
   - For n_projected=3: `np.einsum('ia,jb,kc,ijk', P, P, P, int_nums)` -> scalar-like
   - For n_projected=2: `np.einsum('ia,jb,ijk', P, P, int_nums)` -> 1D
   - For n_projected=1: `np.einsum('ia,ijk', P, int_nums)` -> 2D
   - Docstring cites arXiv:2212.10573 Section 2 (projected volumes)

2. `find_minimal_N(X, epsilon=1e-4, max_val=10000)`:
   - For N in range(1, max_val+1), check if all entries of N*X are within epsilon of an integer
   - Return first such N; raise ValueError if none found
   - No equation citation needed (pure utility)

3. `matrix_period(M, max_iter=200)`:
   - Compute M, M^2, M^3, ... until M^k = I (within tolerance 1e-8)
   - Return k, or raise ValueError if period exceeds max_iter
   - No equation citation needed (pure utility)

4. `get_coxeter_reflection(divisor, curve)`:
   - Formula: `I - 2 * np.tensordot(curve, divisor, axes=0) / (curve @ divisor)`
   - If `curve @ divisor == 0`, return `np.eye(h11, dtype=float)`
   - NOTE: tensor product is `curve (x) divisor` (outer product order matters! — see Pitfall 2 in RESEARCH.md)
   - Docstring cites arXiv:2212.10573 Eq. (4.6) with LaTeX: `M_{ab} = \delta_{ab} - 2 \frac{\mathcal{C}_a D_b}{\mathcal{C} \cdot D}`

5. `coxeter_matrix(reflections)`:
   - Multiplies all reflection matrices: `result = reflections[0] @ reflections[1] @ ...`
   - Uses `functools.reduce(np.matmul, reflections)`
   - Returns identity if empty list
   - Docstring cites arXiv:2212.10573 Section 4 (Coxeter element)

Add `import functools` at top of util.py. Add `from scipy.linalg import null_space` if not already imported (needed later by classify.py but good to have).
