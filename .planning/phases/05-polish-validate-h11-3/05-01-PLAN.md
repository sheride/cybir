**CalabiYauLite.__repr__ (per D-10):**
Compute h11 from `self._int_nums.shape[0]`. If h11 <= 3: show label, h11, flattened unique
intersection numbers (use `int_nums[np.triu_indices(h11)]` or similar compact form showing
the nonzero entries), c2 values, and number of Kahler cone rays (if kahler_cone is set, show
`len(self._kahler_cone.rays())`). If h11 > 3: show only `CalabiYauLite(label=..., h11=N)`.

**CalabiYauLite.__str__:**
Always show the detailed version (intersection numbers, c2, Kahler rays count) regardless of h11.
For h11 > 10, truncate intersection numbers to first 20 nonzero entries with "..." suffix.

**ExtremalContraction.__repr__ (per D-12):**
Show: contraction type display_name (paper notation), contraction curve as list of ints,
zero_vol_divisor as list of ints (if not None), gv_series (if set and h11 of curve <= 10,
i.e. len(contraction_curve) <= 10). Format example:
`ExtremalContraction(symmetric flop, curve=[0,1,-1], zvd=[1,-1,0], gv=[480, 480, ...])`

**CYBirationalClass.__repr__ (per D-11):**
Show: total phases (from `self._graph.num_phases`), fundamental domain phases
(total minus len of `self._weyl_phases`), whether orbit expansion has run
(`self._weyl_expanded`), Coxeter type if available. Example:
`CYBirationalClass(12 phases (4 fundamental), orbit=A2 |W|=6)`
If not constructed: `CYBirationalClass(empty)`.
If constructed but no orbit: `CYBirationalClass(4 phases, no orbit expansion)`.

Also add same guard at the top of `setup_root` method on CYBirationalClass.

<task type="auto">
  <name>Task 2: Stability check mode for adaptive GV degree</name>

  <action>
**Stability check (per D-01, D-02, D-03):**

Add `validate_stability=False` parameter to `construct_phases` in both `build_gv.py` and `ekc.py`.

In `build_gv.py::construct_phases`, after the main BFS loop completes (after the existing
`while True` loop that handles adaptive degree), if `validate_stability=True`:

1. Save a snapshot of the current results: `snapshot_phases = ekc._graph.num_phases`,
   `snapshot_inf = frozenset(ekc._infinity_cone_gens)`,
   `snapshot_eff = frozenset(ekc._eff_cone_gens)`,
   `snapshot_refs = frozenset(ekc._coxeter_refs)`.

2. Bump degree by `deg_step` (up to `max_deg_ceiling`). If already at ceiling, log warning
   "Cannot validate stability: already at max_deg_ceiling" and return.

3. Recompute GVs at new degree (same pattern as existing adaptive bump code).

4. Clear graph and re-run full BFS (same pattern as existing restart).

5. Compare: if `ekc._graph.num_phases == snapshot_phases` AND
   `frozenset(ekc._infinity_cone_gens) == snapshot_inf` AND
   `frozenset(ekc._eff_cone_gens) == snapshot_eff` AND
   `frozenset(ekc._coxeter_refs) == snapshot_refs`, log
   "Stability check passed: results unchanged at deg N+step" and return.

6. If different, log warning "Stability check FAILED: results changed at deg N+step
   (phases: X->Y, inf_gens: A->B, eff_gens: C->D). Keeping higher-degree result.
   Consider running with higher max_deg."

Keep `max_deg=4` as default (per D-03). The stability check is opt-in (per D-02).

Also wire `validate_stability` through `CYBirationalClass.construct_phases` and
`CYBirationalClass.from_gv` (as an additional keyword parameter).
