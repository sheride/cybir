**Detection logic (per D-17):**
In `cybir/core/build_gv.py`, add a helper function `_check_nongeneric_cs(ekc, result)`:
- Takes the classification result dict (which has "contraction_type", "zero_vol_divisor", etc.)
- If `result["contraction_type"]` is `ContractionType.SYMMETRIC_FLOP` AND the result has
  a `"zero_vol_divisor"`:
  1. Get the GLSM charge matrix from the root phase: `root = ekc._graph.get_phase(ekc._root_label)`,
     then `charges = root.charges`. If charges is None, return the result unchanged.
  2. Convert zero_vol_divisor to integer array: `zvd = np.round(np.array(result["zero_vol_divisor"])).astype(int)`
  3. Check if zvd (or -zvd) matches any column of the charge matrix. "Matches" means
     `np.allclose(zvd, col)` or `np.allclose(-zvd, col)` for any column col of charges.
     The GLSM charge matrix has shape (h11, n_divisors); each column is a divisor.
     Actually, `charges` from CYTools is `cy.glsm_charge_matrix(include_origin=False)` which has
     shape `(num_points, h11+1)`. The user said "check if its zero-vol divisor is a prime toric
     divisor (i.e., matches a column of the GLSM charge matrix)". The relevant comparison is
     whether the zero-vol divisor (in the h11-dimensional basis) matches a column of the
     GLSM charge matrix restricted to the h11 basis. Use `root.charges` which was set during
     setup_root from `cy.glsm_charge_matrix()`. Each ROW of charges corresponds to a point/divisor.
     So check if zvd matches any ROW (not column) of charges[:, :h11] or however charges is stored.

     **Ask the user if unclear on the exact shape/convention.** The key idea: for each row of the
     GLSM charge matrix (each prime toric divisor), check if the zero-vol divisor is proportional
     to that divisor restricted to the basis. Use `np.allclose(zvd / np.linalg.norm(zvd),
     row / np.linalg.norm(row))` with appropriate zero checks, or simpler:
     check if zvd is a scalar multiple of any row.
  4. If match found: re-tag `result["contraction_type"] = ContractionType.SU2_NONGENERIC_CS`

- Call `_check_nongeneric_cs` inside `_run_bfs` right after classify_contraction returns a result,
  before `_accumulate_generators`.

**Exclude from orbit generators (per D-18):**
In `_accumulate_generators`, the existing code only adds to `_sym_flop_refs` and `_sym_flop_pairs`
when `ctype == ContractionType.SYMMETRIC_FLOP`. Since re-tagging changes the type to
`SU2_NONGENERIC_CS`, these walls will naturally NOT be added to sym_flop_refs/pairs.
Verify this is the case. SU2_NONGENERIC_CS should be treated like SU2 for generator
accumulation: add coxeter_ref to `_coxeter_refs`, add zvd to `_eff_cone_gens`. Add a new
block in `_accumulate_generators`:
```python
if ctype == ContractionType.SU2_NONGENERIC_CS:
    cox_ref = result.get("coxeter_reflection")
    if cox_ref is not None:
        ekc._coxeter_refs.add(tuplify(np.round(cox_ref).astype(int)))
    zvd = result.get("zero_vol_divisor")
    if zvd is not None:
        ekc._eff_cone_gens.add(tuple(np.round(zvd).astype(int).tolist()))
```

**Create tests/compare_orbit.py (per D-04, D-05):**

Follow the pattern of tests/compare_bfs.py. The script:

1. Sets up sys.path for cornell-dev imports (same 3 paths as compare_bfs.py).
2. Imports `ExtendedKahlerCone` from `extended_kahler_cone` and `CYBirationalClass` from cybir.
3. For each test polytope:
   a. Run original with `ignore_sym=False`: `ekc_orig.construct_phases(weyl=False, ignore_sym=False, verbose=False)`
      This does brute-force BFS through symmetric flop walls, producing the FULL birational geometry.
   b. Run cybir fundamental domain: `ekc_cybir = CYBirationalClass.from_gv(cy, max_deg=max_deg, verbose=False, gvs=orig_gvs)`
   c. Run cybir orbit expansion: `ekc_cybir.apply_coxeter_orbit(phases=True)`
   d. Compare:
      - Total phase count: `ekc_orig.num_phases` vs `ekc_cybir._graph.num_phases`
      - Infinity cone gens (as plain int tuple sets)
      - Effective cone gens (as plain int tuple sets)
      - Coxeter refs (as plain int tuple-of-tuple sets)
4. Accept `--h11 N` (default 3) and `--limit N` (default all) command-line args via argparse.
5. Use the `_to_plain_set` and `_to_plain_set_of_tuples` helpers from compare_bfs.py (copy them).
6. Write results to `tests/compare_orbit_results.jsonl` incrementally.
7. Filter to only polytopes that have symmetric flops (check if `ekc_cybir.sym_flop_refs` is non-empty
   after fundamental domain construction, before running orbit expansion). Skip polytopes without
   symmetric flops (they have nothing to validate).
8. Also skip the non-favorable polytope (check `polytope.is_favorable('M')`).
9. Print summary at end: N polytopes tested, N match, N mismatch.

Use `max_deg=10` for comparison (high enough to be reliable).
